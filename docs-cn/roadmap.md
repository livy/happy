## 工作区和检出

缺少跨越多个机器的**工作区**（也称为项目）概念，以及作为守护进程管理实体的**检出**概念。

- 工作区 = 可以跨越多个机器的逻辑项目（例如我的笔记本电脑 + 云开发机器都在同一个 repo 上工作）
- 检出 = 特定机器上的特定工作副本，由守护进程管理
- 目前我们有机器和路径，但没有一流的工作区分组
- 守护进程应该管理检出生命周期：创建工作树、切换分支、清理陈旧的检出
- 右侧面板上下文（更改、文件）是每个检出的，但工作区跨机器分组检出

## 推送通知路由

当前状态：服务器为每个账户存储裸露的 Expo 推送令牌，没有设备元数据。所有注册设备都会收到所有通知 — 没有路由智能。

- 智能路由：使用在线状态/活动信号（最后活跃设备、哪个设备打开了会话）将通知路由到正确的设备，而不是轰炸所有设备
  - 服务器已经有在线状态数据（sessionCache、机器连接状态）— 可能足以做出好的路由决策
  - 抑制发起操作的设备上的通知
  - 优先选择用户当前活跃的设备
- Web 推送通知：目前完全缺失 — 添加服务工作线程 + Web 推送注册，以便浏览器会话也能收到通知
- 令牌注册时的设备元数据：将平台（ios/android/web）、设备名称、最后活跃时间戳与推送令牌一起存储
