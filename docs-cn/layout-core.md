# Happy 布局 — 核心规范

## 当前布局（来自代码）

使用 `expo-router/drawer` 的双列布局，在平板/桌面设备上使用 `drawerType: 'permanent'`。

### 关键文件

- `SidebarNavigator.tsx` — 带永久侧边栏的 Drawer
- `SidebarView.tsx` — 左列（头部 + SessionsList + FABWide）
- `SessionView.tsx` — 主要内容包装器
- `ChatHeaderView.tsx` — 绝对定位的头部（web 上 maxWidth 800）
- `AgentContentView.tsx` — ChatList + AgentInput
- `ChatList.tsx` — 反转的消息 FlatList
- `AgentInput.tsx` — 编辑器（web 上 maxWidth 800）
- `ActiveSessionsGroupCompact.tsx` — 紧凑的会话行（56px，按项目分组）
- `layout.ts` — maxWidth 常量（web: 800, mac: 1400）

### 设计决策

**两边侧边栏宽度相同（约 300px）。** 中心保持视觉居中。在 1440px 时，中心约为 840px — 自然接近当前的 800px maxWidth，使其变得多余。去掉聊天内容和头部的 maxWidth 约束；让它们填充中心列。

**主要内容应可调整大小。** P1 拖动左边缘以加宽/收缩中心。
**侧边栏也应可调整大小** P2

**上下文面板是每个工作区的。** 在同一工作区中切换会话，右侧面板保持不变。更改、文件 — 都限于工作区。

**禅模式是唯一的切换。** 两个侧边栏在桌面上始终可见。`Cmd+0` 或禅模式按钮隐藏两者，中心变为全宽。没有单独的面板切换。
禅模式会切换掉更多！它应该像 Bear  markdown 编辑器一样 — 完全专注于内容 — 0 干扰

**禅模式按钮位于中心头部。** 紧挨着会话头像，中心列的右上角 — 不在窗口边缘。

### 中心头部行

```
[←] 会话标题                              [●32 头像] [◎ 禅模式]
    ~/path/to/project
```

- `[←]` 返回按钮（现有）
- 标题 + 副标题（现有）
- `[●32]` 带弹出菜单的会话头像（现有）
- `[◎ 禅模式]` 禅模式切换（新）

### 活动会话中的 AgentInput（编辑器）

```
┌──────────────────────────────────────────────┐
│ 在此输入...                                    │
└──────────────────────────────────────────────┘
[⚙ 设置][⏹ 中止] [git±]  [⬆]
```

- `[⚙]` 打开设置覆盖层（权限模式、模型、工作量）
- `[⏹]` 中止当前操作
- `[git±]` git 状态徽章 — 打开文件查看器
- `[⬆]` 发送按钮（32x32，右对齐）

## 上下文面板（右侧）

### 子标签

顶部有两个标签：**已更改** | **全部**

#### 已更改（首先构建）

工作区的 Git 差异文件列表。带行数的已修改/添加/删除文件。

```
M src/auth.ts          +3  -2
M src/routes.ts        +1  -1
A src/helpers/token.ts  +12
```

点击文件 → 中心列变为统一差异查看器，滚动到该文件。

#### 全部（稍后）

完整的分层文件树浏览器。只浏览 - 但在打开文件时 - 应该能够编辑文件。
只在桌面编辑！

#### 期望的（不在 v1 中，不在标签中）

每个人都喜欢 arc ... chrome 正在引入垂直标签 ... 但 conductor 正在将行业推回水平。很高兴为您带来缓解！

在标签内容下方，未来添加：
- **重要标签** — 代理最近读取/写入的文件的"工作集"骨架
- **引导式游览** — 入门讲解入口点，灵感来自 [Graphite Code Tours](https://graphite.com/blog/code-tours)
- **管道 / 流程** — ACP 循环、自定义管道、CI 状态
- **终端** — 机器上的活动终端，带端口 - 甚至可能打开它们

## 差异查看器（中心）

在上下文面板中点击文件时，中心用单个可滚动的差异表面替换聊天。

```
┌──────────────────────────────────────────────────────────────────────────────────────┐

│ │                ││ ← 返回聊天          差异（3 个文件 +16-3）│                    ││
│ │ （不变）       ││                                            ││ [已更改]    [全部] ││
│ │                ││ ┄┄ src/auth.ts ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  ││                    ││
│ │                ││（向上滚动查看）                           ││  src/auth.ts  +3-2 ││
│ │                ││                                            ││ ▶src/routes.ts+1-1 ││
│ │                ││ ┄┄ src/routes.ts ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  ││  src/helpers  +12  ││

│ │                ││  51 │ import { newAuth } from './auth'     ││                    ││
│ │                ││  52 │                                      ││                    ││

│ │                ││     │+  app.use(newAuth)                   ││                    ││
│ │                ││  54 │                                      ││                    ││

│ │                ││  2 │+   return Date.now() > token.exp      ││                    ││

```

- 所有更改文件的单个连续可滚动表面（如 GitHub PR"文件已更改"）
- 文件头部作为粘性部分分隔符
- 滚动时，上下文面板索引高亮显示当前文件
- 点击索引中的文件 → 平滑滚动到该部分
- 折叠的未更改区域（点击展开）
- `← 返回聊天` 返回到相同滚动位置的会话
